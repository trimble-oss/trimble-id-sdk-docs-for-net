# Contributing

Thank you for your interest in contributing to the Trimble Identity SDK documentation and samples for .NET. This guide explains how to participate effectively.

## Code of conduct

This project follows our [Code of Conduct](./CODE_OF_CONDUCT.md). By participating, you agree to uphold it.

## Security

Do **not** open public issues for security vulnerabilities. Follow the process in [SECURITY.md](./SECURITY.md).

## Questions and feedback

- **Bugs and feature requests:** Use the [issue tracker](https://github.com/trimble-oss/tcp-sdk-docs-for-net/issues).
- **General support:** See [README.md](./README.md#support) for contact options.

## Pull requests

1. **Fork the repository** and create a branch from the default branch with a short, descriptive name (for example, `fix/desktop-guide-typo` or `add-maui-sample-note`).
2. **Describe your change** in the pull request: what problem it solves, what you changed, and how you verified it (for example, building a sample or proofreading rendered Markdown).
3. **Keep scope focused.** Prefer smaller, reviewable changes over large mixed refactors unless discussed with maintainers first.
4. **Link related issues** when applicable (for example, `Fixes #123`).

Maintainers are listed in [MAINTAINERS.md](./MAINTAINERS.md). Reviews are handled by the [code owners](./.github/CODEOWNERS) for the areas you touch.

## What to contribute

- **Documentation:** Content under `docs/` (core, desktop, MAUI) and release notes under `release-notes/`. Match the tone and structure of existing guides and FAQs.
- **Samples:** Code under `samples/`. Keep samples minimal, buildable, and consistent with the corresponding documentation.

When editing Markdown, use clear headings, consistent lists and tables, and follow the existing file layout. When editing C# samples, follow common .NET conventions and match patterns used in neighboring projects.
